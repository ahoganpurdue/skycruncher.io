# Solver capability registry

This registry inventories, capability by capability, whether a piece of legacy solver code was
merely present, actually selected onto the executed path, or provably engaged in producing a
result — the three are easy to conflate and this document exists to keep them separate (see
below). Status: **COLD-PATH RECORD** — a Phase 0 artifact from an approved remediation plan
([SOLVER_ARCHITECTURE_AUDIT_AND_TARGET_DESIGN_2026-07-20.md](../04-research/SOLVER_ARCHITECTURE_AUDIT_AND_TARGET_DESIGN_2026-07-20.md)).
The inventory below was the Phase 0 snapshot of the legacy TS/WASM solver taken to plan the
cutover; the cutover itself has since landed and the greenfield Rust solver core is now the
shipping engine. Everything catalogued here describes the frozen legacy lane (`rest/wt-quadonly`),
retained for cold-path reference — never deleted. Doc-only; this file makes no code change and
authorizes none.

**Why three columns, not one.** The audit's own executive finding: *"'Implemented,' 'selected,' and 'engaged' must become three separate receipt fields. Source presence is not execution evidence."* (audit, §"Wired, branch-only, and unwired inventory"). A capability can be implemented and never selected, selected and never engaged, or engaged in a test harness and never engaged in production. Collapsing the three into one status word is exactly the drift this registry exists to stop. Each column below is answered independently:

- **Implemented** — the code exists, cited `file:line@commit`. Presence only.
- **Selected-by** — the flag/config/condition that puts this code on the executed path, plus its actual resolved default (not the value a nearby comment claims).
- **Engaged-proof** — the specific receipt field or counter whose presence/value is the only valid evidence this code actually ran and affected an outcome. Where no such field exists, the cell reads `NO ENGAGEMENT PROOF` — never a guess.

**Authority (Phase 0 formalization, superseded by the cutover).** The legacy solver lane of record was **`rest/wt-quadonly`** — all legacy-track solver surgery landed there while `main` stayed frozen for solver work. Cutover has since landed on `cutover/greenfield-seam`: the greenfield Rust solver core, built on its own track (`greenfield/solver-core`, per the standing internal design brief for that engine), is now the shipping engine. `rest/wt-quadonly` and every other pre-cutover solver worktree/branch stay FROZEN AS EVIDENCE — cold path, retained, never deleted.

**Commit pins.** The main-tree citations below were captured at commit `88d87212`, the Phase 0 main HEAD before cutover (contains only two doc commits beyond the addendum's verification pin `e2120130` — `git diff --stat e2120130 88d87212` touches `CLAUDE.md` and two `docs/` files only, zero source, so code citations verified at `e2120130` held unchanged at `88d87212`). Cutover has since landed on `cutover/greenfield-seam` (starting `d5cfd7b6`) — main's HEAD has moved past this pin, so `88d87212` is no longer current; re-verify any main-tree citation against the live tree before acting on it rather than treating this pin as still-current. Rows marked **[BRANCH]** exist only on `rest/wt-quadonly@386d636b` (VF chain `d9b1ed47→c7473e91→11540e9c→386d636b`; read via `git show rest/wt-quadonly:<path>`) and were **absent from main** at that pin — both the `88d87212` main pin and `rest/wt-quadonly` are now historical/cold-path references from the Phase 0 inventory, retained as evidence.

**Sourcing discipline.** Every cell is either a direct code citation from this pass, or traced to one of: the audit, the [verification addendum](../04-research/SOLVER_AUDIT_VERIFICATION_ADDENDUM_2026-07-20.md) (trusted over the audit on every point of conflict — it verified ~30 claims same-day, refuting 2 and confirming/amplifying most others), `CLAUDE.md`'s 2026-07-20 state stamp, or the Test 1 / Test 1d / conformance-check receipts under `test_results/mag15_build_2026-07-19/`. Where none of these establish a cell, it reads `UNVERIFIED` or `NO ENGAGEMENT PROOF` rather than an assumed value.
