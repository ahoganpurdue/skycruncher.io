# Image & Signal Conditioning — algorithm card (demosaic, binning, background, VST, stretch lane)

Symbols: (x,y)=pixel; ADU=raw sensor counts (u16); "normalized" = (v−black)/(white−black), clamped ≥0, unitless (nominally [0,1] — see UNIT WARNINGS: the demosaic lane has no upper clamp); α=Poisson slope (ADU/e⁻), σ=read-noise-equivalent std in ADU; MAD=median absolute deviation; c_j/w_j=starlet smooth/detail planes.
Every entry below is VERIFIED against the code at the cited line; none are assumed from the paper alone.

### 1. Bilinear Bayer demosaic (WGSL compute, RGGB) + parameterized-CFA sibling
- **Kernel** (per site class, 3×3 neighbors n/s/e/w/ne/nw/se/sw around center c): R site → r=c, g=(n+s+e+w)/4, b=(nw+ne+sw+se)/4; B site → mirror; G-on-R-row → r=(w+e)/2, b=(n+s)/2; G-on-B-row → r=(n+s)/2, b=(w+e)/2. Border pixels (row/col 0 and last) are skipped, output stays 0.
- **Calibration**: out_c = max(0, (v_c − BLACK)·WB_c/(WHITE−BLACK)) with **HARDCODED** `BLACK_LEVEL=2048.0`, `WHITE_LEVEL=16383.0` (14-bit Canon), `WB_R=2.1, WB_G=1.0, WB_B=1.4` — `demosaic_bayer.wgsl:63-69`, flagged `TODO: Phase 4: Pass these via Uniforms` at `:64`.
- **Input packing**: raw is u16 pairs little-endian inside u32 words; `readU16(idx)` = `(raw[idx>>1] >> ((idx&1)*16)) & 0xFFFF` (`:31-37`) — `idx` is an ELEMENT index, not bytes. Input indexed by `stride`, output tightly packed by `width`, 3 f32/px (`:50,108`). Workgroup 16×16.
- **Sibling**: `demosaic_bayer_param.wgsl` — identical kernel, but a 48-byte uniform carries `cfa_offset_x/y`, `black_level`, `white_level`, `wb_r/g/b` (`:21-34`); R sits where `((x+off_x)&1)==0 && ((y+off_y)&1)==0`, so RGGB→(0,0), GRBG→(1,0), GBRG→(0,1), BGGR→(1,1) (`:18-19,78-79`).
- **Code**: `src/engine/pipeline/m3_gpu_preprocess/shaders/demosaic_bayer.wgsl:39-113`; sibling `shaders/demosaic_bayer_param.wgsl:49-116`; CPU twin with the same defaults (`?? 2048/16383/2.1/1.0/1.4`) at `demosaic_engine.ts:128-193` (`:179-183`).
- **Cite**: Bayer CFA — Bayer (1976), US Patent 3,971,065; bilinear interpolation is the baseline method of the demosaicing literature, surveyed in Gunturk, Glotzbach, Altunbasak, Schafer & Mersereau (2005), "Demosaicking: Color Filter Array Interpolation", IEEE Signal Processing Magazine 22(1), 44.
- **Gotcha**: the legacy shader is `include_str!`'d by the native path (`src-tauri/native_gpu/src/demosaic.rs:11`) with a 16-byte uniform and MUST NOT change — that is why the param variant is a separate file, not an edit (`demosaic_bayer_param.wgsl:5-9`). WB gains are applied INSIDE the demosaic, before any stretch — this RGB proxy is white-balanced while the luminance bin (entry 2) is not; the two are not photometrically comparable.

### 2. 2×2 Bayer→luminance binning (WASM) + CFA phase-spread classifier
- **Bin formula**: out[y,x] = clamp( (p₀₀+p₀₁+p₁₀+p₁₁ − 4·black) / (4·(white−black)), 0, 1 ), block anchor `r_idx = 2y·stride + 2x`; output dims w=⌊width/2⌋, h=⌊height/2⌋ (integer division). An unweighted box MEAN — for RGGB the "luminance" is (R+2G+B)/4, no photometric weighting.
- **Classifier** (pattern-agnostic, runs BEFORE binning): per-phase means m₀..m₃ over every 2×2 block (full scan, no subsampling — a strided sample can alias the parity, `demosaic_engine.ts:33-35`); `phaseSpreadL0 = (max(mᵢ)−min(mᵢ)) / max(µ−black, 1e-6)`; `phaseSpreadL1` = same statistic recomputed on the 2×2-binned image indexed by the block's own parity (i.e. the original 4×4 super-phase). Decision: L0 ≥ 0.035 → `std-bayer`; else L1 ≥ 0.10 → `quad-bayer` (supported=false); else `mono`. Thresholds `PIPELINE_CONSTANTS.CFA_CLASSIFY_L0_THRESHOLD=0.035` / `CFA_CLASSIFY_L1_THRESHOLD=0.10` (`src/engine/pipeline/constants/pipeline_config.ts:276-277`), calibrated on a real Canon 60Da Carina frame: measured L0=0.060–0.077, L1=0.0003 (`:273-275`). Frames smaller than one 2×2 block classify as mono with an explicit reason string (`demosaic_engine.ts:60-66`).
- **Routing** (`src/engine/pipeline/m4_signal_detect/signal_processor.ts:159-184`): std-bayer → true luminance bin; mono → same kernel, honestly relabelled "2×2 box-average luminance (no CFA weighting applied)" (`:168-172` — correct because the kernel is unweighted); quad-bayer → NO debinner exists, proceeds on the best-effort bin FLAGGED-UNRELIABLE with `supported=false` on the packet (`:161-167,189`).
- **Code**: WASM kernel `src/engine/wasm_compute/src/lib.rs:718-754`; JS wrapper + web-target WASM-init dance `src/engine/pipeline/m3_gpu_preprocess/demosaic_engine.ts:207-252`; classifier `:44-115`.
- **Cite**: the 2×2 "super-pixel" CFA collapse is standard amateur-astro practice (e.g. Siril's super-pixel debayer) [VERIFY — tutorial/vendor-grade, not peer-reviewed]; the L0/L1 phase-spread classifier is an in-house statistical test — no external source (--).
- **Gotcha**: `bin_bayer_to_luma`'s `black_level`/`white_level` are **u16** at the FFI boundary (`lib.rs:723-724`) — a fractional ADU pedestal truncates silently; the TS wrapper pulls floats from `PhotometryManager.getProfile()` (`demosaic_engine.ts:216-217`). Black/white must be set BEFORE binning (forensic calibration strip ingested first, `signal_processor.ts:137-144`) or the bin clips with default levels.

### 3. Three-space coordinate ledger (native / science / preview)
- **Spaces**: NATIVE = 1:1 sensor px; SCIENCE = 2×2-binned, `scienceW=⌊sensorW/2⌋`, `scienceH=⌊sensorH/2⌋` (`scale_manager.ts:29-31`); PREVIEW = `⌊sensorW·s⌋×⌊sensorH·s⌋`, s=min(1, maxPreviewDim/sensorW), maxPreviewDim default 1920 (`:33-36`).
- **Coordinate maps**: scienceToNative multiplies by the RATIO `nativeW/scienceW` (`:43-49`) — ≈2 but NOT exactly 2 when the native dimension is odd (⌊5185/2⌋=2592 ⇒ ratio 2.00039); hard-coding ×2 would drift half a pixel across the frame. Same ratio form for nativeToscience/nativeToPreview/previewToNative (`:51-73`).
- **Plate-scale maps** (arcsec/px): `getscienceScale = nativeScale·(nativeW/scienceW)` — the plate scale roughly DOUBLES after binning (`:77-80`); generic `getBufferScale(nativeScale, bufferWidth) = nativeScale·(nativeW/bufferWidth)` and inverse `getNativeScale` (`:87-95`).
- **Letterbox**: `scale=min(canvasW/previewW, canvasH/previewH)`, offsets `ox=(canvasW−previewW·scale)/2` (`:100-105`); canvasToNative = previewToNative((cx−ox)/scale, (cy−oy)/scale) (`:111-120`); nativeToCanvas is the exact inverse composition (`:126-136`).
- **Code**: `src/engine/pipeline/m2_hardware/scale_manager.ts:11-149`. Downstream re-projection example: detections found in science space restore to native via `scienceToNative` and FWHM scales by the same `nativeW/scienceW` ratio (`signal_processor.ts:192-204`).
- **Cite**: in-house coordinate convention — no external source (--).
- **Gotcha**: `PIXEL_SIZE_MM = 0.00376` (3.76 µm) is a hardcoded class constant labelled "Default for many Sony/Nikon sensors" (`:12`) — any physical-size math using it is wrong for other pitches unless overridden. Preview scale is derived from WIDTH only (`:34`) — fine while aspect is preserved, but `ui_scale_factor = previewW/nativeW` (`:38`, duplicated at `:146`) silently assumes the height obeyed the same factor.

### 4. Background: 2nd-order polynomial surface fit + global iterative sigma-clip estimate
- **Surface model**: z = A·x² + B·y² + C·xy + D·x + E·y + F, fit by normal equations (XᵀX)β = XᵀZ over sky samples, solved with a 6×6 Gaussian elimination (pivot guard |pivot|<1e-18 → abort, keep zeros) (`TerrestrialEnvironment.ts:104-163,175-195`). Sampling: 24×16 grid cell centers; cells containing any detected star are masked out; cells below the 160-column horizon silhouette or within 20 px above it (`hIdx=⌊x/(w/160)⌋`) are skipped; needs ≥10 surviving samples or the fit is silently skipped (`:109-140`). Evaluation `evaluate(x,y)` returns the modeled sky at any pixel (`:168-170`).
- **Global estimate** (WASM `estimate_background_wasm`): subsample `step_by(4)`; per iteration — sort, median = middle element, σ = √(mean((v−median)²)) (RMS deviation about the MEDIAN — not MAD, not stdev-about-mean), clip to [median±3.0σ], stop early if nothing was clipped; 3 iterations, returns [median, σ] (`src/engine/wasm_compute/src/statistics.rs:13-61`). TS wrapper with defaults iterations=3, sigma_clip=3.0 at `src/engine/core/StatisticsProvider.ts:14-30`.
- **Code**: `src/engine/pipeline/m4_signal_detect/TerrestrialEnvironment.ts:93-190`; global `StatisticsProvider.ts:14-53`; consumers `src/engine/pipeline/m4_signal_detect/source_extractor.ts:47-53,115`, fit call `signal_processor.ts:305-306,365`, subtraction consumer `src/engine/pipeline/m8_photometry/BackgroundExtractionEngine.ts:27-56` (`result = max(0, lum − model.evaluate(x,y))`, then black-floor `getBlackLevel()/65535` subtraction).
- **Cite**: iterative κ·σ-clipped background/noise estimation follows the practice standardized by Bertin & Arnouts (1996), "SExtractor: Software for source extraction", A&AS 117, 393 (lineage for the clipped estimate only — their mesh+spline background differs); the star/horizon-masked polynomial surface is an in-house gradient model (--). **deg-2 is a deliberate ceiling (project LAW)**: deg-3 "ate the galactic band" — a cubic surface absorbs real extended nebulosity/Milky-Way flux as background — so the surface model is fixed at 2nd order. Do NOT raise the polynomial degree.
- **Gotcha (four found)**: (a) the WASM comment "1/16th of pixels" (`statistics.rs:22`) is stale — `step_by(4)` on the flat array samples 1/4; (b) the CPU fallback estimates σ via MAD·1.4826 (`StatisticsProvider.ts:41-44`) while the WASM primary uses RMS-about-median — the two paths return DIFFERENT σ on the same data (MAD is outlier-robust, RMS is not); (c) the surface fit RUNS every frame (`signal_processor.ts:365`) but its model is never consumed — `BackgroundExtractionEngine.applyCorrection` has no callers (the `signal_processor.ts:36` import is unused) and `bgModeler` is never `.evaluate()`d, so gradient subtraction is currently dormant; (d) **arg-slot bug — FIXED 2026-07-10 (commit cf62a1b), retained for lineage**: `source_extractor.ts:47-58`'s `estimateBackground` previously forwarded `(data, width, height)` into the `(iterations, sigma_clip)` slots, so `sigma_clip ≈ image-height` disabled clipping and the shipped estimate was a single-pass UNCLIPPED median + RMS-σ, NOT the documented 3-iteration 3σ clip — silently desensitizing the detector while it was live. The wrapper now takes `(data)` only and calls the calibrated `StatisticsProvider.estimateBackground(data, 3, 3.0)` defaults (in-code `// BUGFIX:` comment), so the live path DOES perform the documented 3-pass/3.0σ clip. Also `TerrestrialEnvironment.ts:135`'s "Sample local median to avoid isolated hot pixels" is a stale comment — the code samples a SINGLE pixel.

### 5. GAT (generalized Anscombe VST) + EXACT-UNBIASED inverse
- **Forward**: GAT(y) = (2/α)·√(α·y + 0.375·α² + σ²) on offset-subtracted y; argument ≤ 0 clamps the output to 0 (`denoise.mjs:206-209`). Whole-plane wrapper subtracts `model.offset` first, NaN→NaN (`:244-252`).
- **Exact-unbiased inverse** (the shipped one): σ_n² = (σ/α)²; domain floor D_min = 2√(0.375 + σ_n²), d = max(D, D_min); u = d²/4 + (√1.5)/(4d) − 1.375/d² + 0.625·√1.5/d³ − 0.125 − σ_n²; return α·max(u,0), offset restored by the plane wrapper (`:217-230,255-263`).
- **Naive inverse (the TRAP, kept only as the bias demonstration)**: u = D²/4 − 0.375 − σ_n² (`:237-241`) — biases low counts downward; NOT used in the pipeline, exists so the fixture can measure the bias the exact form removes (file header `:18-22`).
- **Noise-model source**: FITS metadata gain/read-noise in e⁻ when real (`source:'FITS_META'`, α=1/gain, σ=rn·α, `:88-102`), else photon-transfer estimation (entry 6) labelled `APPROXIMATE`. VST validity floor `lowCountThresh=20` counts/px; >50% of pixels below it sets `low_count.regime_flag=true` + an APPROXIMATE note inside the receipt's `low_count` block (the `noise_model.label` APPROXIMATE/MEASURED flag is separate and unaffected) (`:46,359-367,441-448`).
- **Code**: `tools/denoise/denoise.mjs:199-263` (constants/defaults `:42-47`; pipeline use `:355-418`).
- **Cite**: Anscombe (1948), "The transformation of Poisson, binomial and negative-binomial data", Biometrika 35, 246; generalized (Poisson–Gaussian) form — Starck, Murtagh & Bijaoui (1998), *Image Processing and Data Analysis: The Multiscale Approach*, Cambridge Univ. Press; exact unbiased inverse — Mäkitalo & Foi (2013), "Optimal Inversion of the Generalized Anscombe Transformation for Poisson-Gaussian Noise", IEEE Trans. Image Processing 22(1), 91.
- **Gotcha**: render-layer ONLY (two-ledger law, header `:6-10`) — measurement never runs on denoised data. This lane works in RAW ADU with its own `offset` pedestal, unlike entries 1-2 which emit normalized [0,1]; feeding a normalized buffer here makes the photon-transfer α meaningless. A ZWO-style "GAIN" register value is NOT e⁻/ADU and must not be passed as metadata gain (`:85-87`).

### 6. Starlet à trous decomposition + per-scale κ·σ_j soft-threshold + photon-transfer noise model
- **Transform**: B3-spline kernel [1,4,6,4,1]/16, separable, à trous dilation step 2^(j−1), mirror-reflect boundary; detail w_j = c_{j−1} − c_j, coarse residual c_J kept; reconstruction is exactly additive data = c_J + Σw_j (`denoise.mjs:32,268-332`).
- **Per-scale noise**: σ_j = σ·s_j with `STARLET_NOISE_FACTORS = [0.8907, 0.2007, 0.0857, 0.0413, 0.0205, 0.0103, 0.0052]` (Starck–Murtagh 2-D table, `:34-40`); beyond the table s_j keeps halving (`:390`). VST-domain σ is measured, not assumed: σ_noise = MAD(w₁)/0.6745 / 0.8907, and ≈1.0 is the built-in health check (`:383-387,436`).
- **Shrinkage**: t_j = κ·σ_j with κ=3.0 default (`:43`); soft threshold sign(w)·max(|w|−t, 0) on detail scales only, 5 scales default, coarse residual untouched (`:334-340,381-396`). Optional bounded detail re-injection DEFAULT OFF (`:45,398-410`).
- **Noise-model estimation** (structure-robust photon transfer): Var = α·(mean−offset) + σ² is read off the FINEST wavelet scale, not raw tile variance — per 32×32 tile (>75% finite, strict inequality), noiseSig = (MAD(w₁ tile)/0.6745)/0.8907 (i.e. `madSigma` already folds in the 0.6745 normalization), local mean = median of the SMOOTHED plane; offset = 5th-percentile tile mean; points binned into 12 mean-quantiles, per-bin MEDIAN variance, least-squares line on the bin medians → α, σ²; degenerate fit (α≤0, e.g. read-noise-dominated stacks) falls back to α=1e-6 + MAD-Gaussian σ², flagged low-confidence (`:104-197`).
- **Code**: `tools/denoise/denoise.mjs:29-197` (transform `:268-332`, shrink+pipeline `:334-418`).
- **Cite**: starlet/à trous — Starck, Murtagh & Fadili (2010), *Sparse Image and Signal Processing: Wavelets, Curvelets, Morphological Diversity*, Cambridge Univ. Press (per-scale noise table); à trous origin — Holschneider, Kronland-Martinet, Morlet & Tchamitchian (1989), in *Wavelets: Time-Frequency Methods and Phase Space*, Springer; soft thresholding — Donoho (1995), "De-noising by Soft-Thresholding", IEEE Trans. Information Theory 41(3), 613; photon-transfer method — Janesick (2007), *Photon Transfer: DN → λ*, SPIE Press [book, not a journal paper].
- **Gotcha**: the coarse residual preserves LARGE-scale extended flux exactly, but fine diffuse structure rides the shrunk detail scales — an adversarial review measured ~9% diffuse-flux loss at ~1σ/px, documented in the code itself (`:348-353`); do not cite this lane as flux-preserving. MAD here uses /0.6745 (`madSigma`, `:65-74`) — same constant as the ×1.4826 form used elsewhere in the codebase, inverse notation.

### 7. STF/MTF auto-stretch + two-part color calibration + luminance-ratio chroma scaling
- **MTF**: MTF(x,M) = ((M−1)·x) / ((2M−1)·x − M), clamped MTF(≤0)=0, MTF(≥1)=1; M solved so the calibrated luminance median lands at target B: M = mnorm·(B−1)/(2·B·mnorm − B − mnorm) (`ImageProcessor.ts:193-199`). **LIVE `B = 0.15`** (`:193`) — the method docstring still says "target B=0.25" (`:117`); the inline comment owns the drift ("0.25 read washed-gray"). LINKED stretch: one c0/M for all three channels (`:121-124`).
- **Shadows clip**: c0 = clamp(median − 2.8·1.4826·MAD, 0, 1); mnorm = (median−c0)/(1−c0); degenerate frames (mnorm ≤ 1e-6 or > 0.45) fall back to plain 1/2.2 gamma (`:174-183,236-241`). Stats from a ~80k-pixel subsample, stride bumped +1 when it divides the row width to avoid parity aliasing (`:128-131`).
- **Two-part color calibration** (`:141-166`): (1) subtractive background neutralization — per-channel background medians shifted to the common pedestal = min(med_R, med_G, med_B); (2) highlight white balance — per-channel gain = spanRef/span_c where span_c = (98th percentile of the sub-0.95 population) − med_c, clamped to [0.25, 8]; combined transform cal(v,c) = pedestal + (v−med_c)·gain_c. Rationale in code: a bright-star ensemble is statistically near-white (`:147-151`).
  - **SPCC-WB override hook (LANDED 2026-07-10, `282a423`/`d7e6214`)**: `ImageProcessor.ts:133-202` now takes an optional `spccWb:{gains,applied}` arg; when `applied`, the catalog-derived **TLS channel gains REPLACE** the empirical highlight-white-balance gains here (same `[0.25,8]` clamp, `:199-200`) — with no applied gains the render is byte-identical (`:142-143`). A body color matrix (`camera_color_matrix.ts`) rotates cam-RGB→linear-sRGB BEFORE this step. Full estimator + N=7 gate: CARD_COLOR_PHOTOCAL #8–#9.
- **Chroma scaling** (`:205-225`): L = 0.2126R + 0.7152G + 0.0722B (Rec. 709 weights); Ls = MTF((L−c0)/range); ratio = Ls/L; background-desat weight wSat = smoothstep(0.85·B, 2.2·B, Ls); highlight-rolloff weight wHi = smoothstep(0.85, 1.0, Ls); out_c = Ls + wSat·(c·ratio − Ls), then out_c += wHi·(Ls − out_c) — sky fades to neutral, saturated cores clip toward white.
- **Code**: `src/engine/core/ImageProcessor.ts:110-227`; consumers `src/engine/pipeline/orchestrator_session.ts:363,423`.
- **Cite**: the MTF rational form and STF auto-stretch convention originate in PixInsight (Pleiades Astrophoto; Siril implements the same function) [VERIFY — vendor documentation, not peer-reviewed]; MADN constant 1.4826 = 1/Φ⁻¹(3/4) — Hampel (1974), "The Influence Curve and its Role in Robust Estimation", JASA 69, 383; luma weights — Rec. ITU-R BT.709.
- **Gotcha**: B=0.25 in the docstring vs B=0.15 live is real document-drift inside the shipped file — trust `:193`. Per-channel MTF was tried and rejected ("olive soup", A/B note `:185-187`); do not "fix" the linked stretch back to per-channel. The 98th-percentile star-white gains exclude saturated pixels (≥0.95) BEFORE taking the percentile (`:156-161`) — reordering that filter changes the white point.

## UNIT WARNINGS
- **Two pedestal regimes coexist**: entries 1-2 emit NORMALIZED output with black already subtracted at source (`demosaic_bayer.wgsl:101-105`, `lib.rs:747`) — but only the bin clamps to [0,1] (`lib.rs:749`); the demosaic clamps ≥0 with NO upper clamp, so WB gain 2.1 can push bright pixels to ~2.1 (`demosaic_bayer.wgsl:103-105`). The denoise lane (entries 5-6) works in RAW ADU with its own fitted `offset`. Never feed one lane's buffer into the other's math without checking which pedestal was already removed. `BackgroundExtractionEngine.applyCorrection:48-53` subtracts `getBlackLevel()/65535` from a buffer whose pedestal was already removed at bin time — AND the /65535 assumes 16-bit full scale on a 14-bit (16383) sensor. Currently dormant (no callers) but a live trap if wired up.
- **FFI truncation**: `bin_bayer_to_luma` takes black/white as u16 (`lib.rs:723-724`); fractional ADU levels truncate silently at the WASM boundary.
- **arcsec/px is per-BUFFER**: science plate scale = native × (nativeW/scienceW) ≈ 2× (`scale_manager.ts:77-80`). Quoting a native plate scale against science-space FWHM/positions is this lane's equivalent of the RA hours-vs-degrees trap.
- **Binned ratios are not exactly 2**: science dims are ⌊native/2⌋; all cross-space maps use the ratio `nativeW/scienceW`, never a literal 2 (`scale_manager.ts:43-57`) — keep it that way for odd sensor dimensions.
- **WGSL raw indexing**: the storage buffer packs two little-endian u16 per u32; `readU16` takes an element index, not a byte offset (`demosaic_bayer.wgsl:31-37`).
- **WB asymmetry**: the demosaic RGB proxy is white-balanced (2.1/1.0/1.4); the binned luminance is NOT — brightness comparisons across the two proxies are invalid.
