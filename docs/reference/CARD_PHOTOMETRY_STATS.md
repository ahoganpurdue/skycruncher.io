### 1. Sigma-threshold detection + culling-reason tally
- **Formula**: coarse mask threshold = mean + k·σ (global stats over sampled luminance), k=`sigFactor`=2.0. Downstream: an independently-tuned cascade of per-blob SNR/circularity/flux gates (LOW_SNR if bg≈black-floor & snr<15; circularity<0.8 reject only if snr<75; saturated-star void carve if circ>0.85 & snr>100; planet routing if snr>1200 or ephemeris-locked; hot-pixel anomaly if snr>300) — NOT one global threshold.
- **Code**: `src/engine/pipeline/m4_signal_detect/signal_processor.ts:49-53` (coarse mean+kσ), `:512-550` (per-reason cuts), `:654,727,739,750,768` (SNR/flux gates); tally reconciliation in `culling_stats.ts:32-63` (`computeCullingCounts` — never invents a "dropped" count when no pre-tally exists).
- **Cite**: generic sigma-clipped background thresholding; the canonical astronomical implementation of this pattern is SExtractor (Bertin & Arnouts 1996, A&AS 117, 393), but this is an independent from-scratch cascade, not a SExtractor port or reimplementation.
- **Gotcha**: the file's own header docstring claims "Thresholding (**5**-sigma)" (`:12`) — the LIVE coarse constant is **k=2.0σ** (`:51`). The 5σ figure is stale/aspirational documentation, not the shipped value; real star/junk separation happens in the downstream per-reason cuts, not the coarse mask.

### 2. Per-star PSF fit — Levenberg-Marquardt, 2D Gaussian
- **Model**: F(x,y) = A·exp(−(a·dx²+b·dx·dy+c·dy²)), dx=x−cx, dy=y−cy; a,b,c = standard rotated-Gaussian coefficients from (σx,σy,θ). Update: (JᵀJ+λI)δ=Jᵀr; λ×10 on singular solve, λ×0.1 on success. FWHM = 2√(2·ln2)·σ ≈ 2.3548σ.
- **Code**: `src/engine/wasm_compute/src/photometry.rs:9-152` (`fit_gaussian_2d`), `:158-219` (`refine_stars_bulk_impl`, per-star stamp extraction); caller `src/engine/pipeline/m10_psf/psf_field.ts:39` (FWHM constant), `:12-19` (fitter contract + honest degrade to moment fallback when no compiled fitter is present).
- **Cite**: Levenberg (1944), Quart. Appl. Math. 2, 164; Marquardt (1963), J. Soc. Indust. Appl. Math. 11(2), 431. Profile confirmed **Gaussian** — no Moffat β-profile anywhere in this file/module (grep-confirmed); Moffat (1969) does NOT apply here.
- **Gotcha**: no gain-ratio / chi-square-decrease test gates step acceptance — `photometry.rs:137` comment states "Skipping rigorous chi-square check for raw performance"; every step with an invertible (JᵀJ+λI) is accepted unconditionally, only λ's direction depends on invertibility. This is a deviation from textbook LM's trust-region acceptance rule. F also carries NO pedestal/background term — the caller must background-subtract each stamp first (`psf_field.ts:14,371`), else the fit is biased.

### 3. Richardson-Lucy deconvolution (damped, nebulosity-protected)
- **Base update**: est ← est · corr(obs / conv(est,K), K) (standard multiplicative RL). This code additionally: dampens the ratio smoothstep-toward-1 within 1.5·σ_noise of unity, clamps ratio∈[0.25,4], freezes saturated pixels to the observed value every iteration, and (protected variant) splits the window into a box-blurred DIFFUSE component + a COMPACT residual so RL only sharpens the compact part.
- **Code**: `src/engine/pipeline/m10_psf/rl_deconv.ts:180-230` (`richardsonLucyWindow`, damping `:200-211`), `:154-177` (`richardsonLucyWindowProtected`, diffuse/compact split), `:105-134` (`boxBlur`).
- **Cite**: Richardson (1972), "Bayesian-Based Iterative Method of Image Restoration", JOSA 62(1), 55; Lucy (1974), "An iterative technique for the rectification of observed distributions", AJ 79, 745.
- **Gotcha**: NOT vanilla RL — damping band, ratio clamp, saturated-core freeze, and diffuse/compact splitting are all local additions, explicitly labeled APPROXIMATE (`PsfReport.approximate`). Windowed (bounded local stamps, not full-frame) with clamp-to-edge convolution; default OFF; runs on the NATIVE-grid PIXEL ledger only, never touches WCS. **UPDATE (2026-07-10, merge `10b51e7`)**: the nebulosity-**protected** variant (`richardsonLucyWindowProtected`) is now the WIRED live path behind the default-OFF flag — still gated, still APPROXIMATE, still subordinate to the render-ML beautification ruling (gated / AESTHETIC-labeled / DEFAULT-OFF / subordinate to the forced-photometry preservation gate / C2PA-documented). Known artifact stands: RL rings on ultra-deep stacks (annular ringing — see this card's annular note below).

### 4. Forced photometry at catalog positions + scrambled-null escalation
- **SNR**: snr = flux / noise; flux = apSum − n_ap·bg_median; noise = σ_local·√(n_ap + n_ap²/n_bg) — aperture Poisson-like term PLUS the propagated uncertainty of the annulus background mean itself. Background: median + MAD·1.4826 sigma-clip (one 3σ clip round) over annulus [r_ap+3, r_ap+8], widened until ≥40 px.
- **Escalation statistic**: binomial-excess z = (observed − n·p0) / √(n·p0·(1−p0)), p0 = max(nullRate, 0.5/nullN); null calibrated from `scrambles`(=5) draws of deterministic (mulberry32 PRNG) uniform on-frame positions through the SAME aperture/threshold as the real measurement.
- **Code**: `src/engine/pipeline/m6_plate_solve/deep_verify.ts:91-136` (`forcedMeasure`, noise formula `:134-135`), `:120-130` (sigma-clip), `:207-214` (`sampledBackgroundSigma`), `:271-` (mulberry32), `:335-341` (`binomialExcessZ`), `:368-428` (`deepVerifyEscalation`).
- **Cite**: aperture-noise form is a restricted case of the standard CCD aperture-photometry noise equation (cf. Howell, *Handbook of CCD Astronomy*, 2nd ed., ch. 4 "CCD equation") — background + background-uncertainty terms only, no explicit source-Poisson/dark-current/read-noise terms (folded into the measured σ_local). The binomial-excess z is a standard one-proportion z-test (normal approximation to the binomial), applied here as a bespoke on-frame empirical-null calibration rather than a named detection statistic.
- **Gotcha**: MAD→σ constant 1.4826 assumes a Gaussian background (standard practice, unstated in-code). The scrambled null MUST reuse the identical aperture/threshold as the real measurement for the "chance rate" to be apples-to-apples — changing one side without the other silently breaks the statistic.

### 5. Set-level confirmation gate ("family-wise", actually bespoke)
- **Formula**: setExcessZ = binomialExcessZ(realConfirmed, N, nullConfirmRate, nullTotal) — the SAME one-proportion z-test as #4, run ONCE over the whole candidate set vs. a frame-wide scrambled-null confirm rate (measured from `setNullDraws`(=4) draws through the IDENTICAL per-star confirm predicate: SNR pass + neighbor pass + shape/local-null not-FAIL). Pass ⟺ z ≥ setExcessGateZ AND realConfirmed > expected.
- **Code**: `src/engine/pipeline/m6_plate_solve/forced_confirm.ts:362-428` (`confirmForcedSet`), shared statistic at `deep_verify.ts:335-341`; live threshold wired at `solver_entry.ts:2528` (`setExcessGateZ: PC.SOLVER_CONFIRM_SET_EXCESS_Z`).
- **Cite**: none — this is BESPOKE. Despite the name "family-wise gate" (and in-code comments invoking "family-wise excess"), there is no Bonferroni / Holm / Šidák / Benjamini-Hochberg (FDR) correction anywhere in the file — it is a single aggregate binomial-proportion z-test comparing the real set's confirm rate to an empirically-measured on-frame null rate, not a classical multiple-comparisons family-wise-error-rate procedure. Naming it "family-wise" is a code-comment idiom, not a statistical claim to take literally.
- **Gotcha**: two default values coexist — the module's own `DEFAULT_CONFIRM_CONFIG.setExcessGateZ=10` (`forced_confirm.ts:129`) vs. the ONLY live caller's override to `PC.SOLVER_CONFIRM_SET_EXCESS_Z=15` (calibrated — see docs/GATES.md; **N=1-calibrated, SeeStar-only**, not yet a validated science-product threshold). Reading this file in isolation understates the real live gate by Δz=5.

**"Annular" — two distinct meanings across the cards (disambiguation for fan-out):** (1) **annulus background** = the sky ring for aperture photometry: median (+ MAD·1.4826 one-round 3σ clip) over an annulus outside the aperture — #4 here (`deep_verify.ts`, ring `[r_ap+3, r_ap+8]` widened until ≥40 px) and CARD_COLOR_PHOTOCAL #3 (SPCC RGB photometry, per-channel MEDIAN sky ring `r2..r3`). Robust background estimator, NOT an artifact. (2) **annular ring ARTIFACT** = the ringing RL deconvolution introduces on ultra-deep stacks — #3 here, a KNOWN failure mode the damping/protected variant mitigates, default-OFF. When a fan-out says "annular", pin which one: the cards default to meaning (1) unless the context is RL/deconvolution.
