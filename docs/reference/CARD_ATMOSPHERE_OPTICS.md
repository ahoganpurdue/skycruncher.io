# Atmosphere & Optics Physics — algorithm card (extinction, refraction, PSF-predictor lane)

Symbols: h=apparent altitude (DEGREES), z=zenith distance (90−h), X=airmass, τ=zenith optical depth, Δm=extinction (mag), λ=wavelength, D=aperture diameter (mm), FL=focal length (mm), f#=focal ratio, δ=Dec, RA in **HOURS** unless stated, ″=arcsec, ′=arcmin, r=normalized radial coordinate.
Every entry below is VERIFIED against the code at the cited line; formulas transcribed from the running code, not idealized from the literature.

### 1. Kasten & Young (1989) airmass — plus the COMPETING plane-parallel sec z model in the PSF lane
- **Formula (extinction lane)**: X = 1 / (sin(h_rad) + 0.50572·(6.07995 + h)^−1.6364). Caps: h ≤ 0 → X = 40 (below-horizon cap, includes h=0 exactly); h ≥ 90 → X = 1.0.
- **Formula (PSF lane, competing)**: X = 1 / max(0.05, cos z) — plane-parallel sec z with the cosine clamped at 0.05, i.e. an implicit cap of X = 20.
- **Code**: `src/engine/core/AtmosphericManager.ts:25-32` (`computeAirMass`; cap `:26`, formula `:31`); variant `src/engine/pipeline/m10_psf/psf_physics.ts:162-165` (`airmassFromAltitude`, self-labeled "APPROXIMATE below ~30°"). PSF consumer additionally nulls airmass when altitude ≤ 3° (`src/engine/pipeline/stages/psf_attribution.ts:355`).
- **Cite**: Kasten & Young (1989), "Revised optical air mass tables and approximation formula", Applied Optics 28(22), 4735.
- **Gotcha**: TWO airmass models coexist for the same sky with different saturation behavior (K&Y cap 40 vs sec-z cap 20 vs PSF-consumer null below 3° alt) — CLAUDE.md's "Kasten & Young airmass" convention describes only the extinction lane. Also a house unit trap baked into K&Y's own published form: the polynomial term `(6.07995 + h)^−1.6364` takes h in **DEGREES** while `sin` takes RADIANS — a well-meaning "convert everything to radians" refactor silently breaks the formula.

### 2. Allen zenith Rayleigh optical depths + Δm = τ·X + zenith flux multipliers
- **Constants**: τ_R = 0.044 (comment: ~620 nm), τ_G = 0.098 (~530 nm), τ_B = 0.235 (~450 nm) — `AtmosphericManager.ts:17-19`, in-code attribution "Allen's, 4th ed."
- **Formula**: per-channel extinction Δm = τ·X (`rayleighExtinction`, `:37-43`); restoration multipliers m_c = 10^(0.4·Δm_c) (`getZenithMultipliers`, `:49-56`).
- **Application**: `src/engine/pipeline/m5_coordinate_flatten/zenith.ts:276-292` (`applyZenithCorrection`) — the pixel is converted sRGB→CIE XYZ, then the RGB-keyed multipliers are applied on the TRISTIMULUS axes: X ← X·(m_R+m_G)/2, Y ← Y·m_G, Z ← Z·m_B, then XYZ→sRGB back.
- **Cite**: values consistent with the Rayleigh-scattering optical depths in *Allen's Astrophysical Quantities*, 4th ed. (Cox, ed., 2000) as the in-code comment claims [VERIFY — reference volume, not checked against the printed table this pass].
- **Gotcha (documented drift, ×2)**: (a) the τ→mag factor **1.0857** (= 2.5·log₁₀e) named in CLAUDE.md is ABSENT from shipped code — grep of `src/` finds zero hits; the live form Δm = τ·X runs a factor 1.0857 (~7.9%) BELOW the physically-correct Δm = 1.0857·τ·X (= 2.5·log₁₀e·τ·X). Tracked as an unharvested fix from remote commit d865aab in `docs/NEXT_MOVES.md:40` ("Rayleigh 1.0857 factor"). (b) the docstring at `zenith.ts:282` claims "Work in linear light (remove gamma) → correct → reapply gamma", but the correction is applied in CIE XYZ, not linear RGB — X is not the R primary (hence the (m_R+m_G)/2 hack); this is the same NEXT_MOVES line's "zenith linear-RGB" item. Cross-lane note: the green channel is keyed to ~530 nm here but λ_g = 0.55 µm in the diffraction lane (entry 5).

### 3. Bennett refraction + field differential refraction (REVIVED predictor-only consumer)
- **Formula**: R = 1 / tan(h + 7.31/(h + 4.4)) ARCMINUTES at standard pressure/temperature (in-code: 1010 hPa, 10 °C); h clamped ≥ 0.01° to dodge the horizon singularity; returned ×60 as ARCSECONDS. Differential: ΔR(h₁,h₂) = R(h₁) − R(h₂).
- **Code**: `src/engine/core/optics_manager.ts:290-298` (`calculateAtmosphericRefraction`), `:304-308` (`getDifferentialRefraction`). Wrapper: `src/engine/pipeline/m5_coordinate_flatten/differential_refraction_corrector.ts:23-41` (thin delegation). LIVE consumer: `src/engine/pipeline/stages/psf_attribution.ts:375-391` — gated on trusted clock AND real GPS (`gpsReal && clockReal`, `:347-349,376`), computes the plate stretch across the vertical field: altTop = min(89.9, h + halfFOV), altBot = max(0.1, h − halfFOV), halfFOV = (imageHeight/2)·pixelScale/3600.
- **Cite**: Bennett (1982), "The Calculation of Astronomical Refraction in Marine Navigation", Journal of Navigation 35(2), 255. The formula formerly quoted in the wrapper header was Sæmundsson's (now disclaimed there) — Sky & Telescope 72, 70 (1986) [VERIFY — hobbyist-magazine source, NOT the shipped formula; see Gotcha].
- **Gotcha (header RESOLVED 2026-07-09 — retained for lineage)**: the wrapper header was previously stale on two counts and is now FIXED (commit 39c3d8d, the same commit that added this card). It now reads "STATUS: LIVE as an APPROXIMATE, clock+GPS-gated field-level PREDICTOR" (`differential_refraction_corrector.ts:11-21`) and explicitly disclaims the old "R = 1.02·cot(h_a + 10.3/(h_a + 5.11))" Sæmundsson form, using the true Bennett 7.31/4.4 form the executing code runs. (Historically the header claimed "UNWIRED / INERT — this class is not imported by any pipeline stage" — FALSE since the revival, `psf_attribution.ts:38` imports it and `:380` calls `computeDifferential` every gated run — and mislabeled the Sæmundsson form as Bennett; both corrected.) Still PREDICTOR ONLY: the result feeds the PSF report, never the solve chain (`:389` note); the chromatic single-star dispersion term is explicitly DEFERRED (achromatic model can't yield it).

### 4. Seeing airmass scaling θ = θ_zenith·(sec z)^0.6
- **Formula**: θ = θ_zenith · max(1, X)^0.6 arcsec, with θ_zenith = `DEFAULT_SEEING_ZENITH_ARCSEC` = 2.0″ — an ASSUMED amateur-site constant, never measured.
- **Code**: `src/engine/pipeline/m10_psf/psf_physics.ts:51` (constant), `:138-140` (`seeingArcsec`); consumer `psf_attribution.ts:358-373` (tier `APPROXIMATE`, falls back to X=1 when no observing geometry, and pushes the assumption into the report's `approximate[]` ledger).
- **Cite**: the 0.6 = 3/5 exponent follows from Kolmogorov turbulence, r₀ ∝ (cos z)^(3/5) — Fried (1966), JOSA 56(10), 1372; the 2.0″ zenith constant has no source ('--', assumption).
- **Gotcha**: the airmass fed in is the PSF lane's plane-parallel sec z (entry 1 variant), NOT Kasten & Young — near the horizon the two lanes disagree about X and therefore about predicted seeing. `max(1, X)` silently floors nonsense sub-1 airmass instead of rejecting it.

### 5. Chromatic diffraction floor: Airy FWHM 1.028·λ/D + Rayleigh first null 1.22·λ/D
- **Formula**: θ_FWHM = 1.028·(λ/D)·(648000/π)″ with D = FL/f# (mm) and λ in mm (input µm × 10⁻³); θ_Rayleigh = 1.22·λ/D (reported for reference only). Per-channel λ = {r: 0.62, g: 0.55, b: 0.47} µm. Arcsec-native, grid-independent — the caller divides by pixel scale to reach science-grid px.
- **Code**: `src/engine/pipeline/m10_psf/psf_physics.ts:43,45,48` (constants `DIFFRACTION_FWHM_COEFF`, `RAYLEIGH_COEFF`, `LAMBDA_UM`), `:124-128` (`diffractionFwhmArcsec`), `:130-133` (`rayleighArcsec`); consumer `psf_attribution.ts:325-344` (tier `CALCULATED`, `limitedGreen` flag = measured minPx ≤ 1.3× green floor).
- **Cite**: Airy (1835), "On the Diffraction of an Object-glass with Circular Aperture", Trans. Camb. Phil. Soc. 5, 283; Rayleigh (1879), Phil. Mag. 8, 261 (resolution criterion). The 1.028 FWHM coefficient of the Airy core is textbook optics (Born & Wolf, *Principles of Optics*, §8.6) [VERIFY — textbook constant, not re-derived here].
- **Gotcha**: a FLOOR, never a subtraction — the consumer's note (`psf_attribution.ts:340`) states it explicitly; using it to deconvolve would be a category error. D = FL/f# inherits every upstream focal-length lie (the 50 mm factory-default EXIF trap, `optics_manager.ts:46-61`); when FL/f#/scale are missing the floor is honestly NOT MEASURED (`:343`), not defaulted.

### 6. Sidereal drift + det-sign-aware CD axis directions + parallactic angle + zenith image PA + drift kernel
- **Drift**: trail = 15.041″/s · cos(δ) · t_exposure (`SIDEREAL_RATE_ARCSEC_PER_SEC` `psf_physics.ts:37`, `siderealTrailArcsec` `:76-78`) — constant stored truncated from 360°·3600″/86164.0905 s = 15.04107.
- **Axis directions** (CD in deg/px): RA axis = atan2(−cd₁₀·sgn, cd₁₁·sgn), Dec axis = atan2(cd₀₀·sgn, −cd₀₁·sgn), sgn = sign(det CD) — the det sign encodes parity and is LOAD-BEARING for the full [0,360) direction; it cancels under the [0,180) line fold (`:83-103`). `synthesizeCd(scale, roll, parity)` builds an approximate CD when no fitted one exists, folding parity into the ξ-row sign (`:108-118`).
- **Geometry**: local GMST = (280.46061837 + 360.98564736629·(JD − 2451545.0)) mod 360, JD from Unix epoch (`targetGeometry` `:146-160`, RA input in HOURS, ×15 at `:152`); parallactic angle q = atan2(sin H, tan φ·cos δ − sin δ·cos H) (`:168-172`); zenith image-space PA rotates image North by ±q toward image East with the sense read from the CD handedness (`zenithPaImageDeg` `:181-190`).
- **Drift kernel**: exact uniform 1-D line ("top-hat along the trail") of `lengthPx` at `paDeg` — DESCRIBED only, never deconvolved (`driftPsfKernel` `:303-310`, FOLLOW-ON seam for a known-kernel deblur).
- **Cite**: sidereal day 86164.0905 s per IAU-1982 (Aoki et al. 1982, A&A 105, 359); the linear GMST form and the parallactic-angle formula match Meeus, *Astronomical Algorithms*, 2nd ed., chs. 12 & 14 [VERIFY — practical textbook, not peer-reviewed; the GMST here drops Meeus's T² and T³ terms].
- **Gotcha**: this is a SECOND, inline GMST implementation, independent of `TimeService` (whose IAU-1982 continuous form is the house convention per CLAUDE.md) — it treats UTC as UT1 and omits the quadratic terms; fine for drift PAs, not for precision timing. Untracked-mount assumption: `siderealTrailArcsec` models a FIXED tripod; any tracking (even partial) makes the "exact" trail an upper bound.

### 7. Coma form fit — 1-parameter through-origin least squares e(r) = k·r
- **Formula**: k = Σ(rᵢ·eᵢ) / Σ(rᵢ²) over 3×3 field regions; r = region distance from field center / half-diagonal (normalized [0,1], center region r < 10⁻⁶ skipped — no radial axis). Consistency: median over regions of the line-angle separation between measured major-axis PA and the radial PA (`lineAngleSepDeg`, [0,90]); `patternConsistent` ⟺ k > 0 AND medianDev ≤ 30° (`radialTolDeg` default) AND n ≥ 4 (`minRegions` default). Through-origin R² = 1 − SS_res/SS_tot with SS_tot about the MEAN.
- **Code**: `src/engine/pipeline/m10_psf/psf_physics.ts:227-284` (`fitComaCoefficient`; slope `:256`, R² `:258-272`, median `:274-275`, gate `:276`); consumer `psf_attribution.ts:399-418` (9-region grid, tier `FITTED`, coefficient never fabricated from EXIF).
- **Cite**: the coma FORM (transverse blur growing linearly with field height, flare oriented radially) is third-order Seidel aberration theory — Born & Wolf, *Principles of Optics*, ch. 5 [VERIFY — textbook]; the 1-parameter fit itself is bespoke ('--').
- **Gotcha**: the R² pairs a through-origin model with a mean-centered total sum of squares — statistically inconsistent (standard through-origin R² uses the uncentered SS) and can go NEGATIVE when the fit is worse than the mean; the median of an even-length deviation list takes the UPPER middle element (`devs[len>>1]`). The output field name `coeffPerPx` (and its "per px of field radius" doc, `psf_physics.ts:206`) is a MISNOMER — r is normalized to the half-diagonal, so k is ellipticity per unit NORMALIZED radius (i.e. at the field corner), not per pixel. The fitted ellipticity is the TOTAL measured field — drift/seeing/tilt contaminate k; only the radial-orientation gate separates coma from impostors.

### 8. SECOND Brown-Conrady implementation + vignetting fit + FOV + focal-length recovery
- **Forward (ideal→distorted)**: with normalized (x_n,y_n) = ((x,y) − center)/r_ref, r² = x_n²+y_n²: radial = 1 + k1·r² + k2·r⁴ + k3·r⁶; dx_tang = 2p1·x_n·y_n + p2·(r²+2x_n²); dy_tang = p1·(r²+2y_n²) + 2p2·x_n·y_n; out = center + (x_n·radial + d_tang)·r_ref. r_ref = profile.r_ref, else half-diagonal when width,height > 0, else 1.0.
- **Inverse (distorted→ideal)**: TRUE Newton-Raphson on the radial scalar, 10 iterations: f = r_u(1+k1r_u²+k2r_u⁴+k3r_u⁶) − r_d, f′ = 1+3k1r_u²+5k2r_u⁴+7k3r_u⁶, early-out |f| < 10⁻⁷; then a "simplified" tangential removal — the forward tangential terms are SUBTRACTED, evaluated on a HYBRID: distorted normalized (x,y) but r² = r_u² from the NR result (first-order approximation, code comment admits "p1, p2 are usually tiny").
- **Coefficient solver**: regress y = (r_d/r_u) − 1 on {r², r⁴, r⁶} via 3×3 normal equations solved by Cramer's rule (`solve3x3`, determinant guard 10⁻¹⁵); needs ≥ 6 pairs; p1 = p2 = 0 pinned in the output profile.
- **Vignetting**: factor = Σᵢ coeffs[i]·r^(2i), clamped [0,1]; correction = 1/factor gated factor > 0.01; profile fit = through-origin regression of (intensity − 1) on r² → coeffs [1.0, k1], needs ≥ 5 points.
- **Also here**: FOV = 2·atan(d/(2·FL)) degrees (`getFov`); FL recovery FL_mm = 206.265·pitch_µm/scale_″/px (`recoverFocalLengthFromScale` — 206.265 = 206265 arcsec/rad ÷ 1000 for the µm→mm fold; POST-solve, honest-or-absent).
- **Code**: `src/engine/core/optics_manager.ts:144-171` (`applyDistortion`), `:177-221` (`removeDistortion`; NR loop `:198-208`, tangential removal `:212-219`), `:227-264` (`solveDistortionPolynomial`), `:266-282` (`solve3x3`), `:316-320` (`getFov`), `:328-362` (vignette factor/correction), `:369-383` (`solveVignettingProfile`), `:131-138` (FL recovery). Live callers: `src/engine/core/SkyTransform.ts:66,86`; `src/engine/pipeline/m5_coordinate_flatten/coordinate_flattener.ts:131`; `src/engine/pipeline/m5_coordinate_flatten/generic_flattener.ts:44,58` (thin single-point delegation — live via `stages/detect.ts:74` and `orchestrator.ts:1115`; its BATCH path `flattenPoints` `:65-99` bypasses this class for WASM, see Gotcha).
- **Cite**: Brown (1966), "Decentering Distortion of Lenses", Photogrammetric Engineering 32(3), 444; Conrady (1919), "Decentered Lens-Systems", MNRAS 79, 384. Vignetting: the fit is a bespoke generic radial polynomial — NOT the cos⁴ law despite the interface's `cosPower` field ('--').
- **Gotcha**: DISTINCT from the already-carded radial-only `m2_hardware/lens_distortion.ts` (k1,k2, fixed-point inverse) — this one applies the FULL k1,k2,k3 + p1,p2 model with a true Newton-Raphson radial inverse; and a THIRD Brown-Conrady implementation ships: a Rust WASM port of `removeDistortion` (`src/engine/wasm_compute/src/lib.rs:632`, `flatten_coordinates`), consumed by the batch path `generic_flattener.ts:65-99` (`flattenPoints`, live via `detect.ts:74` / `orchestrator.ts:1115` / `src/engine/tools/verify_pipeline.ts:245`). ## UNIT WARNINGS
- Altitude/h enters EVERY function on this card in DEGREES; radian conversion is internal. Kasten & Young deliberately mixes units (degrees inside the polynomial, radians inside sin) — do not "fix" it.
- TWO airmass conventions: extinction lane caps X=40 below horizon (`AtmosphericManager.ts:26`); PSF lane caps X=20 via the cos clamp (`psf_physics.ts:164`) and its consumer nulls X below 3° altitude.
- Refraction returns ARCSECONDS (`optics_manager.ts:297`, arcmin×60 internally) — Bennett's native published unit is arcminutes; a caller assuming arcmin is 60× wrong.
- Wavelength bookkeeping differs per lane: extinction τs are keyed to nm in comments (620/530/450), diffraction takes µm (0.62/0.55/0.47) — and the GREEN centers disagree (530 vs 550 nm).
- Distortion coefficients are only meaningful WITH their normalization: half-diagonal PIXELS in the flattener lane vs tangent-plane DEGREES (r_ref||1.0) in the SkyTransform lane.
- RA is HOURS at the `targetGeometry` boundary (×15 to degrees at `psf_physics.ts:152`); CD matrices in the PSF lane are DEGREES/px.
- Zenith flux restoration multiplies CIE XYZ tristimulus axes, not linear RGB channels — despite the RGB-keyed multiplier names.
